# Maintainer Guide

This document describes how the public registry is generated and maintained.

## Workflow

Each asset repository declares its metadata in frontmatter:

- `skill-*` repositories use `SKILL.md`
- `agent-*` repositories use `AGENT.md`

The public workflow runs nightly:

1. List public `skill-*` and `agent-*` repositories.
2. Clone each repository read-only.
3. Run `scripts/validate_skill.py`.
4. Generate display-safe public artifacts:
   - `registry.json`
   - `INDEX.md`
   - `llms.txt`
   - `.claude-plugin/marketplace.json`

Private audit details are generated by `quantskills/registry-audit`, not by the public registry.

## Public / Private Split

Public `quantskills/registry` contains display-safe data only:

- asset names and URLs
- project type
- declaration file
- category, tags, platform declarations
- summaries
- validation level
- maintainer type
- last validation date

Private `quantskills/registry-audit` contains:

- full `health_items`
- scan reports
- human review reports
- quarantined details

## Safety Boundary

The registry workflow does not modify asset repositories:

- no PRs
- no branch pushes
- no issue writes
- no topics / description / homepage edits
- no workflow changes in scanned repositories

The only write operation is committing generated public artifacts back to this repository.

## Local Use

```bash
pip install pyyaml requests
python scripts/validate_skill.py /path/to/skill-or-agent-repo
GITHUB_TOKEN=xxx python scripts/build_registry.py --full
```

`GITHUB_TOKEN` only needs read access. Public-only scans can run without a token but may hit lower GitHub API rate limits.
